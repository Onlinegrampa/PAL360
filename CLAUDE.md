## AGENT OPERATING RULES
1. NEVER commit secrets, API keys, or credentials to any file.
2. ALWAYS use environment variables via .env.local (frontend) and backend/.env (backend).
3. NEVER connect to live OLAS system. Use /data/seeds/ mock layer ONLY.
4. ALWAYS enforce Supabase Row Level Security (RLS) on every table.
5. On ANY ambiguity in requirements, STOP and ask — do not assume.
6. Prioritize deterministic behavior: no random UI state, no unpredictable API responses.
7. Self-heal broken builds by reading error logs before retrying — max 3 retry attempts.

## PWA CONFIGURATION
Package: next-pwa (wrapper around Workbox)
Install command: npm install next-pwa

### manifest.json (required fields)
- name: "PAL360"
- short_name: "PAL360"
- start_url: "/dashboard"
- display: "standalone"         # Hides browser chrome on install
- background_color: "#002855"   # PAL Navy
- theme_color: "#002855"
- icons: 192x192 and 512x512 PNG (PAL branded, stored in /public/icons/)

### next.config.js PWA setup
- Enable next-pwa in production only (disable in dev to avoid cache conflicts)
- Cache strategy: NetworkFirst for all API routes (/api/*)
- Cache strategy: CacheFirst for static assets (images, fonts, icons)
- Offline fallback page: /app/offline/page.tsx — render "You are offline" with PAL branding

### Service Worker Behavior
- Pre-cache: all static pages on install
- Runtime cache: /policies, /claims, /products endpoints (stale-while-revalidate)
- Push notifications: OUT OF SCOPE for V1 (add in V2 for premium reminders)

### Mobile UX Requirements (enforced by Frontend Agent)
- Viewport meta tag: width=device-width, initial-scale=1
- All tap targets minimum 44x44px
- Bottom navigation bar on mobile (Dashboard | Claims | Products | Pay)
- No horizontal scroll on any screen at 375px width (iPhone SE baseline)
- Touch gestures: swipe between policy cards on Dashboard
- Loading skeletons on all data-fetching screens (no blank states)

## LOCAL DEV — QUICK START

### One command (auto-detects available ports)
```bash
# macOS / Linux / Git Bash
bash dev.sh

# Windows (cmd / PowerShell)
dev.bat
```

Both servers launch and log to the terminal. `NEXT_PUBLIC_API_URL` is written
to `frontend/.env.local` automatically using whichever port the backend binds.

### Manual (if you need separate terminals)
```bash
# Terminal 1 — Backend (FastAPI + uvicorn)
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Terminal 2 — Frontend (Next.js)
cd frontend
npm install
NEXT_PUBLIC_API_URL=http://localhost:8000 npm run dev
```
